Project overview

Automated weekly review of leading AI publications. The workflow filters noise, summarizes updates by business criteria, and delivers a concise audio briefing with a transcript and citations.

Business impact

Keeps busy executives current by converting ever-changing AI news into short, high-signal audio briefs, reducing screen time and improving retention.

My role: Strategy → architecture → build → test

Designed the agentic flow, implemented summarization prompts, integrated ElevenLabs for narration, and automated the weekly build and publish process.

Scalability

Highly flexible, adjust subject matter, sub-topics, cadence, and episode length for domains like finance, healthcare, education, or policy. Can localize by region, add sponsor reads, or produce multi-language feeds.

Technical details

OpenAI ElevenLabs Python GitHub Actions GitHub Pages RSS / feed.xml
  • Data & inputs – weekly automated collection of seven leading AI publications.
  • Model & processing – OpenAI condenses and structures updates into business-focused summaries.
  • Runtime – automated workflow in GitHub Actions to generate audio briefings and update the podcast feed on schedule.
  • Audio & delivery – ElevenLabs TTS produces MP3 narration with aligned transcripts for reference and QC.
  • Hosting & distribution – MP3s and podcast feed hosted on GitHub Pages for direct access through podcast apps.
  • Versioning & publish – workflow commits audio and metadata updates back to the repository for repeatable releases.

Best experienced with headphones; episodes are ~3–5 minutes.
